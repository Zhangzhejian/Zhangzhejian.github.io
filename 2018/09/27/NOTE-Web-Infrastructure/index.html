  1. system architectures LODE

System architecture styles

System architecture styles are formulated in terms of components, the way that components are connected to each other, the data exchanged between components and finally how these elements are jointly configured into a system.

  • Layered architectures
  • Object-based architectures
  • Data-centered architectures
  • Event-based architectures
Layered Architectures
  • Components are organised in a layered fashion where a component layer $L_i$ is allowed to call components at the underlying layer $L_{i -1}$
  • Widely adopted by communication network community
  • Control flows from layer to layer; requests go down the hierarchy whereas results flow upward
Object-based acchitectures

  • This architecture holds components loosely

  • Each component is defined as an object and components are connected through mechanisms defined in the interface (example, RPC)

Data-centered architecture

Web-based distributed systems are largely data-centric.They rely on shared distributed file systems

Processes communicate through a common passive or active repository. They use shared web-based data services.

Event-based architecture

Processes essentially communicate through the propagation of events, which optionally also carry data. Event propagation is generally through publish/subscribe systems.(进程通过事件的传播来交流，可以携带数据。事件传播是通过发布/订阅系统来实现的)

Processes publish events after which middleware ensures that only processes that subscribe receive them.(只有订阅的进程才能接受到)

Client-server model

A server is a process implementing a specific service. A client is a process that requests a service from a server

Centralised architecture(中心化)
  • Clear distinction between clients and servers
  • A leader monitors the system
  • Fault tolerant and robust systems
Decentralised architecture(去中心化)
  • No monitor to observe processes
  • Useful architecture with reliable underlying(block chain) networks
  • Useful with large number of components in the system

Web proxy caching

  • To improve consistency of the system
  • Client-side caching stores frequently visited pages at browser’s cache.

  • Web proxy is another type of client-side cache that is placed between client and web server.

  • Web proxy can implement a shared cache.

  • Cooperative caching or distributed caching
