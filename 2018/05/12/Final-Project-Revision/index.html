Final Project Revision

Posted on 2018-05-12

Final project revision

This is a blog where I will have a brief introduction to my final year project which cost me a half year. I will complete this blog in three weeks to illustrate ever step to build the mini-cloud infrastructure and the problems I met which might be helpful.

And at the final part, I will try to write some experience I got from the project.

Basic introduction

Project name: A mini-cloud based e-commerce platform for small business

Illustration: use one tower server and 7 Raspberry Pis to establish a mini-cloud infrastructure and then develop an e-commerce platforom for small businee which used to deployed on the infrastructure.

PS: This project focuses more on the hardware part: how to establish a distributed system with high availability and reliability

Specific to Raspberry Pi:

Operation system: 2017-4-vga-raspbian-jessie

CPU: QUAD Core ARMCortex-A7

RAM: 1GB

Micro SD: 64G

Specific of Dell PowerEdge™ T110II Server:

Operating system:Ubuntu Trusty 14.04 (LTS)

CPU: Intel Xeon processE3-1200 product family

RAM:32GB

Mini-cloud Infrastructure

Web Frame Work:

Load Balancer:

Two Raspberry Pis to establish a load balacer cluster with failover function.

Software: keepalived, NGINX

The configuration process: Nginx and keepalived configuration

Business layer:

Two Raspberry Pis to make up a process system

Software: tomcat7

The configuration process: not complete yet

Database Layer:

Three Raspberry Pis to establish a data centre with high availability and reliability by using read-write separation and master-slave warm standby

Software: MySQL 5.5.60

The configuration process: Use MySQL to establish master-slave warm standby and read-write separation

Static server layer

One tower server to store all static files like css,js,html.

configuration: wait for completing…

Web Application Layer

Software part is the worst part in my project. Before design and implementation, requirements are not clear enough and the basic knowledge about the distributed system is not well learnt. After finishing the project when do this revision, I found the web application is just a piece of rubbish that has no single sepcific design for a distributed system. Thus the website is just for proving the whole infrastructure can work well.
