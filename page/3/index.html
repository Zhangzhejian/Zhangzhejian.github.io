Difference between game tree and strategy

A game tree represents all the possible games that can be played according to the rules of the game:

• Every branch is a dispute (sequence of allowable moves).

• The game tree contains all possible disputes.

A strategy for player p is a partial game tree:

• Every branch is a dispute (sequence of allowable moves).

• The tree only branches after moves by p.

• The children of p’s moves are all the legal moves by the other player.

A strategy S for player p is winning if and only if p wins all disputes in S.

Decomposition approach(prefered extension)

Idea is to decompose an argumentation framework into a set of sub-frameworks and compute the semantics of each subframework locally.

We need to perform the decomposition so that the resulting subframeworks respect the dependency of the attack relation and form an acyclic graph(非循环图).

Game theory

Basic knowledge

  • Dominant Strategy
  • Nash Equilibrium
  • Pareto optimal
  • social welfare

Properties of agreement mechanism

  • An agreement mechanism is individually rational if no agent will end up worse off as a result of participating (agents cannot end up with negative expected utility). Otherwise it would not be rational for agents to participate.

  • An agreement mechanism is symmetric if the rules of the game are the same for all participants.

  • An agreement mechanism is fair if participants have equal opportunity to influence the outcome.

  • An agreement mechanism is Pareto optimal if it guarantees a Pareto optimal outcome.

  • An agreement mechanism maximises social utility if it guarantees an outcome that maximises social utility.

  • An agreement mechanism is guaranteed to terminate if there is no way that participants following the protocol can continue to interact indefinitely.

  • An agreement mechanism is stable if a Nash Equilibrium exists.

  • An agreement mechanism is simple if it is easy to work out what the best strategy is.

This is about bargaining for resource division

  • One-to-one protocol.

  • Negotiation takes place in series of rounds.

  • Not guaranteed to terminate

Monotonic concession protocol(单方面让步协议)

Negotiation proceeds in rounds.

On round 1, agents simultaneously propose a deal from the negotiation set.

Agreement is reached if one agent finds that the deal proposed by the other is at least as good or better than its proposal. If each agent’s offer matches or exceeds that of the other agent, then one of the proposals is randomly selected as the agreement deal. If only one proposal exceeds of matches the other’s, then this is the agreement deal.

If no agreement is reached, then negotiation proceeds to another round of simultaneous proposals.

In round u + 1, no agent is allowed to make a proposal that is less preferred by the other agent than the deal it proposed at time u.

If neither agent makes a concession in some round u > 0, then negotiation terminates, with the conflict deal.

Zeuthen strategy

The Zeuthen strategy determines what an agent should do by answering three questions.

  • What should an agent’s first proposal be?

    Its most preferred deal.

  • On any given round, who should concede?

    The agent least willing to risk conflict (if equal risk, flip a coin).

  • If an agent concedes, then how much should it concede?

    Just enough to change the balance of risk.

Deception in TODs

  • Phantom and Decoy tasks

    • Pretending that you have been allocated tasks you have not

  • Hidden tasks

    • Pretending not to have been allocated tasks that you have been

Agent communication

agents are autonomous and self-interested. One agent can’t force another to perform some action

Instead they can perform communicative actions to try to influence other agents.

Speech act theory

Speech act theory treats communication as action. It is a theory of how utterances are used to achieve intentions.

Three aspects of speech acts

  • The locutionary act(语言行为): the act of making the utterance

    • Saying “Please make me some tea”

  • The illocutionary act(言外行为):(intention behind utterance) the conveying of the speaker’s intentions to the hearer

    • The speaker has requested the hearer to make some tea

  • The perlocution(表达效果): the effect of the act

    • The hearer (perhaps) makes some tea.

Speaker makes a locutionary act(语言行为) with hope(言外行为) that the hearer correctly recognises the speaker’s intentions and acts(表达效果) accordingly.

Five types of speech act
  • Representatives(阐述):commit the speaker to the truth of an expressed proposition, e.g. ‘it’s raining’.
  • Directiives(指令): attempt to get the hearer to do something, e.g. ‘please make the tea’.
  • Commissive(承诺): commit the speaker to doing something, e.g. ‘I promise to come to the lecture on Thursday’.
  • Expressive(表达): express some mental state, e.g. ‘thank you!’.
  • Declaration(宣告): entail the occurrence of an action in themselves, so effect some changes in an institutional state of affairs, e.g. declaring war, ‘We the jury find the defendant to be guilty’.

we can in general think of a speech act as having two components:

  • a performative verb (e.g request, inform, promise)
  • Propositional content命题内容(e.g ‘the door is closed’)

Example：

performative = request content = ‘the door is closed’ speech act = ‘please close the door’

performative = inform content = ‘the door is closed’ speech act = ‘the door is closed!’

performative = inquire content = ‘the door is closed’ speech act = ‘is the door closed?’

working together

It is often necessary or preferable for agents to work together in order to achieve their goals.

Since agents are autonomous they make decisions about what to do at run-time, they must be capable of dynamically coordinating their activities and cooperating with others.

Compare this with traditional distributed and concurrent systems, where coordination and cooperation is typically hardwired at design time.

two criteria for assessing how well agents work together.

Coherence(连贯性)

is about how “well the [multiagent] system behaves as a unit along some dimension of evaluation”. 作为一个整体的表现

We can measure it in terms of solution quality, e.g. how efficiently resources are used, conceptual clarity.

Coordination(协调性)

is about “the degree. . . to which [the agents]. . . can avoid “extraneous” activity [such as] . . . synchronizing and aligning their activities”. 对于各自活动的同步和调整

If the system is perfectly coordinated, agents will not get in each others’ way, in a physical or a metaphorical sense.

Contract Net Protocol(very important)

deal with the second stage of CDPS: subproblem allocation

Each node on the network can, at different times or for different tasks, be a manager or a contractor. When a node gets a composite task (or for any reason can’t solve its present task), it breaks it into subtasks (if possible) and announces them (acting as a manager), receives bids from potential contractors, then awards the job.

Stage of CNP:
  1. Recognition
    • Recognize a problem it wants help with
  2. Announcement

  3. Bidding
    • Agents receiving an announcement decide whether they wish to bid for the task.
  4. Awarding
    • Agent that sent task announcement must choose between bids and decide who to award the contract to.
  5. Expediting
    • The successful contractor expedites the task

    • Incompatibility不兼容)
  • Possitive Relationships
    • Requested(agent explicitly asks for help with activities)

Some approaches to coordination:

  • Multi-agent planning

  • Joint intentions (teamwork)

  • Normative multi-agent systems

Multi-agent planning

  • centralised planning for distributed agents

Centralised agent has all information (agents’ private goals, actions available) and uses this to determine efficient plan for each agent.

  • Partial global planning

Agents plan for themselves, but also have partial information about the global plan of all agents.

  • Merging of individual plans

Agents plan for themselves, share their plans with a centralised agent, who merges these to produce an efficient global plan

Joint intentions(teamwork)

Agents in team have collective intention towards the goal and have responsibilities to other agents in the team.

A group of agents have a collective commitment to bringing about some goal G; the motivation for this goal is M.

The termination condition is that it is mutually believed that either:

• G is satisfied;

• G is impossible;

• M is not longer present.

Normative multi-agent system

A norm is an established, expected pattern of behaviour.

Norms affect agent reasoning.

If you’re caught, then violating a norm will lead to a decrease in utility (punishment).

Challenges for normative agent systems.

• Agents need to know the norms.

• Agents need to understand the norms.

• Monitoring needs to be effective.

• Punishments need to be suitable.

Lecture 5: Strategic Reasoning(Game Theory)

Game Theory

Game theory is a framework for analysing interactions between a set of agents.

Key points in game theory:
  • Dominant strategy
  • Nash equilibrium strategy
  • Pareto optimal strategies
  • Strategies that maximise social welfare
Dominant strategies(占优策略)

A strategy S is dominant if no matter what the other participants do, you can’t do any better than play S.

Nash Equilibrium(纳什均衡)

In general, we will say that two strategies $s_1$ and $s_2$ are in Nash equilibrium (NE) if:

  1. under the assumption that agent i plays $s_1$ , agent j can do no better than play $s_2$ ; and

  2. under the assumption that agent j plays $s_2$ , agent i can do no better than play $s_1$ .

  • Not every interaction scenario has a pure strategy NE.
  • Some interaction scenarios have more than one NE.
Pareto optimal Strategy

An outcome is said to be Pareto optimal (or Pareto efﬁcient) if there is no other outcome that makes one agent better off without making another agent worse

Social welfare

The social welfare of an outcome ω is the sum of the utilities that each agent gets from ω

Deductive reasoning(演绎推理) agents

Deductive reasoning = particular symbolic approach where representations are logical formulas and reasoning used is logical deduction(逻辑演绎) (theorem proving).

  • Symbolic AI paradigm

  • Contains an explicitly represented, symbolic model of the world.

  • Makes decisions (for example about what actions to perform) via symbolic reasoning.

Use logic to encode a theory stating the best action to perform in any given situation.

Let:

• ρ be this theory (typically a set of rules)

• Δ be a logical database that describes the current state of the world

• Ac be the set of actions the agent can perform

• Δ |- ρ ϕ means that ϕ can be deduced from Δ using ρ

according to the input of environment and the logical database get the current state. then based on the theory and the current state, the agent get the right decision.

planning agents

Planning agents find a sequence of actions that transforms an initial state into a goal state.

Planning involves issues of both Search and Knowledge Representation.

Reactive agent architectures

Traditional AI View

• Symbolic (internal) representation.

• Manipulate this representation to work out what to do next.

Alternative View

• Symbolic reasoning is very resource demanding.

• Effective behaviour does not need representation and manipulation.

• Intelligent behaviour is a product of the agent’s interaction with its environment.

• Intelligent behaviour emerges from the interaction of various simpler behaviours.
