MDD

Model-Driven Development envisages(设想) development of software systems as a process consisting of:

  • construction and transformation of models
  • Semi-automated generation of executable code from models

Can make production more efficient:

  • Free developers from complexity of implementation details(开发者不用去关注实现的细节)
  • Retaining core functionality of a system despite changes in its technology(技术的变化不会影响核心功能的模型变化)

MDD is especially relevant for internet applications because of rapid changes in web technologies, and similar processes across many internet applications.互联网技术的变化非常快(架构，编程语言等)，并且应用之间有极大的相似之处(涉及到模型的相似)

MDA(Model-Driven Architecture)

Constains:

  • PIM (platform-independent models)

  • PSMs (platform-specific models)

  • Model transformations

PIM

  • Implementation-indepndent constructs
    • Express business rules that are core definition of functionalities of system(表述核心功能的业务逻辑)
    • Reusable across different paltforms
    • Flexible to accommodate enhancements and other backwards-compatible changes(对于之后的优化或者向后兼容的改变很灵活)

PSM

Tailored to specific software platform and programming language and define functionalities of system in sufficient detail that they can be directly programmed from the model.(针对软件平台和语言的设计，定义了详细的功能细节，可以直接被实现)

Model Transformations

Produce a new model from an existing model

  • improve the quality(去除冗余，分解出共同元素)
  • Refine a PIM towards a PSM
  • refine a PSM to an implementation

Includes:

  • introduction of design patterns,or(引入设计模式)
  • elimination of model elements that are not supported by a particular platform(such as multiple inheritanc,or many-many association)(去除特定语言平台不支持的元素，比如多重继承，多对多关联)

Bayesian Decision Theory

Provide a method for making statistically optimal decisions

  • Evidence vector x
  • Classes $\omega_j$

Bayes Decision Rule

  • If $P(\omega_1|x) >P(\omega_2|x) $, class as $\omega_1$
  • Otherwise class as $\omega_2$

Likelihood Ratio Test(only two classes)

The left-hand is the likelihood Ratio

The right-hand is a constant

General use in classification

  • the data is multivariate
    • x is a d-dimensional feature vector
  • There are more than two states of nature(classes)
    • $\omega$ is a finite set {$\omega_1,\omega_2…\omega_c$}
    • we choose the one with the max value of the posterior

Bayes Risk

To deal with the misclassifications.

Loss function

$\lambda(\alpha_i|\omega_j)$

  • States how costly it is to take action $\alpha_i$ if the true state is $\omega_j $

For optimal performance, i.e. minimum loss

  • Calculater $\mathcal{R}(\alpha_i|x)$ for all possible actions, then
  • Choose the action $\alpha_i$ for which $\mathcal{R}(\alpha_i|x)$ is minimum

Density Estimation(估算的是概率密度)

To solve the problem in Bayesian Decision Theory that:

  • $P(\omega_1)$
  • $p(x|\omega_1)$

with the data we have.

Solution is to estimate the probabilities from the data, and this process is called Density Estimation

  • for $p(\omega_j) = \frac{n_j}{n}$
  • for $p(x|\omega_1)$
    • Parametric density Estimation
    • Non-Parametric Density Estimation

Parametric Density Estimation

Assumes the probability density can be described as a mathematical function, typically the Normal Distribution(假设概率密度分部遵从某种数学分布，比如正态分布，然后求分布的参数)

  • for normal distribution
    • Assume $p(x|\omega_i)\sim \mathcal{N}(\mu_i,\Sigma_i) $
  • Need to estimate Parameters of this function
    • use data to estimate $\mu_i$(the mean) and $\Sigma_i$ (the covariance协方差)

Two estimation techniques:

  • Maximum-Likelihood(ML)
  • Bayesian Parameter Estimation

Normal Distribution Example

  • Univariate(1-D): $\mathcal{N}(\mu,\sigma)$

    • $\sigma$ is the standard deviation(标准差)

  • Multivariate (d-D): $\mathcal{N}(\mu,\Sigma)$

    • x is a column vector of length d
    • μ is a column vector of length d
    • $\Sigma$(协方差) is a d$\times$d matrix, with determinant $|\Sigma|$ and inverse $\Sigma^{-1}$

Maximum-Likelihood Estimation

Assume data contains n samples, and drawn independently from a multivariate normal distribution

  • the mean $\mu$

  • The covariance

Limitation

Parametric methods are limited:

  • the distribution can be parameterised by a simple, unimodal, mathematical distribution
  • we know the mathematical function

Nonparametric Density Estimation

Parzen window

Can be used for arbitrary and unknown distributions

  • Window size: h
  • Volume of region: V
  • number of samples in region: k
  • Total number of samples: N

To estimate the probability density in point x (x is a d-dimension vector):

$k(x)$ is the function $K: x \to number \; of \; samples \; in \; region$

From another way to see the function k, this one can be called window function:

The function $K(x,x_i)$ can be regarded as: the contribution of a single sample $x_i$ to the region (x,h). SO the density function can donate the average probability contribution to the region.

Several normal window functions

  • Square window

  • Gaussian window

    the Same as the normal distribution

Conclusion

Only works if we have lots of data available- an unquantifiable amount

$k_n$ Nearest Neighbour

Can avoid problem of unknown best window size.

  • let the volume be a function of the traning data
  • Centre a window about x and let it grow until it captures $k_n$ samples
  • These samples captured are the $k_n$ nearest-neighbours of x

Window size varies with density of data about x:

  • High density
    • Window be small which provide good resolution(分辨率)
  • Low density
    • Window wil be large, but estimated probability will be small
    • The continuity are ensured

Also requires lots of data

Reliable broadcasts

Problems:

  • processes receive messages in different order
  • even different messages
  • and sometimes process crashes

Solution：

Properties of Reliable Broadcast

  • Agreement(一致性)
    • if any correct process delivers m, then all correct processes eventually deliver m
    • 确保所有的进程都会收到同样的信息
  • Validity(有效性)
    • if a correct process executes broadcast(m) then all correct processes will eventually deliver m
    • 一次广播，那么所有的都会收到信息，确保这次广播的是有效的
  • Integrity(真实性)
    • deliver(m) occurs at most once at each correct process, and only if broadcast(m) occurred in some process
    • 只有当信息被广播之后，信息才会被送达且最多一次( 真实性，不存在欺骗，不会存在信息没有广播但是被送达的情况)

Note: a correct process is one that has not failed

Uniform Reliable Broadcast

Uniform broadcasts ensure agreement and itegrity to allow failed processes to recover

Properties of uniform reliable broadcast:

  • Uniform agreement:
    • if any process delivers m, then all correct processes eventually deliver m
  • uniform integrity:
    • deliver(m) occurs at most once at each process, and only if broadcast(m) occurred in some process.

注意和reliable broadcast的区别

Order of messages

  • FIFO
    • Applies messages from a particular process
    • 对于一个进程按一定顺序发出的信息，其他所有进程收到的顺序也得遵从
  • Causal ordering
    • If message B is broadcast from P1 after message A is delivered at P1, then at all other processes, B must be delivered after A is delivered
  • Total ordering
    • the order of message delivery is the same at all processes
    • The transmission of messages forms atomic broadcasts (AB)
    • Atomic broadcast guarantee that:
      • A message is delivered to all processes or to none at all
      • All messages are delivered in the same order to all processes.

Consensus on a global scale

A set of computers want to reach agreement that about a single decision (or data item)

Consideration

  • Safety
    • Everyone should agree and never forget
  • Liveness
    • How many peers must be online before a decision is final

Classical Consensus

  • small scale and up to 1/3 fault tolerance
  • safety and fork-avoidance
  • no financial incentive

Nakamoto Consensus

  • Global Scale and 99%+ fault-tolerance
  • Liveness and fork-tolerance(eventual consistency)
    • all computers will eventually agree on a decision
  • Financial incentive(Profit)
    • Rewarded with coins for participating

Bitcoin is the first consensus protocol that strives for liveness instead of safety.

All financially motivated peers(miners) will eventually agree upon a single blockchain.

Complexity of algorithms

n denotes the size of input

  • Polynomial-time algorithm:
    • The worst-case running time is $\Theta(n^k)$
  • Exponential-time algorithm:
    • The worst-case running time is $\Theta(2^{dn})$ and $\Omega(2^{cn})$

Asymptotic(渐进的) notation $\Omega(n)$ $\Theta(n)$ and $O(n)$

  • $O(n)$: is an asymptotic upper bound
  • $Omega(n)$ is a lower bound

Mathmaticla Definition:

  • if $\lim_{n \to +\infty} \frac{f(n)}{g(n)} = 0$, then f(n) is strictly less than g(n). $f(n) = o(g(n))$
  • if $\lim_{n \to +\infty} \frac{f(n)}{g(n)} = \infty$, then f(n) is strictly larger than g(n). $f(n) = \omega(g(n))$
  • if $\lim_{n \to +\infty} \frac{f(n)}{g(n)} = c $, then f(n) is strictly equal to g(n). $f(n) =g(n)$

P, NP, NP-hard, NP-complete

  • P: problems can be solved in polynomial time
  • NP: ‘N’ denotes “non-deterministic”(非确定). Which meas these problems is non-deterministic can be solved in polynomial time currently(至少现在不确定有没有线性复杂度的算法能解决这些问题), but can be verified in polynomial time given an answer.
  • In order for further definition, we introduce a new definition here. “Reduction(归约，约化)” means a problem A can be reduced from problem B, if A can be transformed to B. Which means B is as hard as A.
  • NP-complete: NPC problem is kind of problem that:
    • is a NP problem(can be verificated polynomially)
    • all NP problem can be reduced from it
  • NP-hard: NP problems are as hard as NP problems
    • all NP problem can be reduced from it
    • but not a NP

Combinitional Optimisation(组合优化)

A combinatorial optimisation problem:

  • view an instance as a pair $(\mathcal(R), \mathcal(C))$
    • $\mathcal(R)$ is a finite set of configurations
    • $\mathcal(C)$ is a cost function, $\mathcal(C): \mathcal(R) \to real \; number$
  • Problem: find a confuguration with the minimum cost. That is to find a configuration config $\in \mathcal(R)$ such that the cost $\mathcal(C)(config)$ of this configuration is as small as possible.
  • Illustrations: if the size of the configuration set $\mathcal(R)$ is N, the the number of config, the subset of $\mathcal(R)$ is $2^N$. So in order to find the minum of the config, the complexity of the problem is $\Omega(2^n)$.
    • NP-hard
  • Examples: TSP, SAT

Design Considerations

Openness(对开发人员)

Interfaces to the OS, network and services must be based on published specifications (such as standards)

  • This makes it possible for one vendor to build systems which interact with those of another
  • Without such openness, the user is in practice restricted to purchase from only one vendor
  • Different users will buy from different vendors, but cannot later build a distributed system

Too many standards can be almost as bad as vendor proprietary standards, since the user must then deal with all the different standards.

Concurrency

Multiple clients may try to access a service.

The whole system is multi-process, so issues related to multi-process programming should be considered:

  • lock
  • Synchronizing

Scalability(伸缩性) and state

Should avoid:

  • the notion of single master services
  • short fixed names(对一些参数的命名，不能太短)
  • performance bottlenecks

Some resources include some sort of state information: data that alters the resources’ future response to requests, e.g.

  • The state of a file server is the data held on disk
  • Alterations to the state alter the response given to a READ request

Duplication of state information leads to the problem of propagating changes of state to the various copies

可以理解为数据的一致性consistency

Fault tolerance

  • Fault isolation: fault occurring in one resource does not effect operation of other resources
  • Fault masking: take action to restore the service when a particular resource of the service fails

SO hardware redundancy and software recovery are required

Measure of failure rate

Mean Time before Failure(MTBF)

  • Average period of time between failures
  • Assume a random distribution of failures
  • Can compute the probability of failure occurring in any period of time

T-fault tolerance

Must be more than t failed components before a service fails

Failure modes and illustrations

  • Failstop:
    • a processor halts, and remain halted. The fact that the processor has halted can be detected by other processors
  • Crash:
    • As for failstop, except not detectable by other processors
  • Crash+link:
    • A link fails for a period, losing some messages(信息传递通道的fail)
  • Receive omission:
    • Processor fail to read message sent to it
  • Send omission:
    • processor fail to write message required by protocols
  • General ommision:
    • receive or send omission
  • Byzantine failure:
    • the processor behaves in a manner not permitted by the protocol

Transparency(对用户)

Transparency is about making the service provided independent from how it is provided.

Types of transparency:

  • Access transparency:
    • Methods to access a resource do not vary according to means of access, e.g. Java method call, SOAP message
  • Location Transparency:
    • Access to resource does not require specifying its location.
  • Concurrency Transparency:
    • Access to a resource is not affected by number of clients
  • Replication Transparency:
    • Methods to access a resource do not change according to how many copies of that resource are maintained
  • Failure Transparency:
    • A service remains available even if a resource which supplies that service fails Migration Transparency: Moving a resource between hosts does not affect access to it
  • Performance Transparency:
    • Increases or decreases in resource performance do not alter methods by which the resource is accessed
  • Scaling Transparency:
    • Access to resources does not change according to overall size of the system or number of instances of resource

Security

  • Preventing actions which by accident prevent the correct functioning of the system
