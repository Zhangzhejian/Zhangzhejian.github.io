Agents with state

Internal data structure state typically used to record information about the environment state and history. $\mathcal{I}$ be the set of all internal states of the agent

The function next is introduced, which maps an internal state and a percept to an internal state, and then the action-selection function action maps an internal state to an action.

Control Loop
  1. Agent starts in some initial internal state $i_0$
  2. Observes its environment state e, and generates a percept $see(e)$
  3. Internal state of the agent is then updated via next function, becoming $next(i_0 , see(e))$
  4. The action selected by the agent is $action (next(i_0 , see(e)))$
  5. Goto 2

Tasks for Agents

How to specify the tasks for agents is a problem

Maxmise utility
Associate utilities with individual states

the task specification is a function which asscociates a real number with every environments state

Disadvantage: difficult to specify a long term view when assigning utilities to individual states

Associate utilities with runs

Example: trading agent might use $u(r) = (p(r) l(r)) x fp(r)$ where $fp(r)$ is profit in the final state of r, $p(r)$ is the number of states agent is in profit and $l(r)$ is the number of states agent is not in profit.

Expected utility & optimal agents

Expected utility

P(r|Ag,Env) to be denoted as probability that run r occurs when agent Ag is placed in environment Env.

Expected utility: $u(r)P(r|Ag,Env)$

Optimal agent $Ag_{opt}$ is the one that maximizes expected utility over all possible runs in the environment

Difficulties with utility-based approaches

  1. Not obvious how to generate the numbers
  2. People don’t think in terms of utilities
  3. Hard to formulate tasks in these terms

Predicate(断言) task specifications

A special case of assigning utilities to runs is to assign 0 (false) or 1 (true). Denote predicate task specification by Ψ.

How to do the right thing

Need to be able to

  1. identify the alternatives

  3. and do so quickly enough for the decision to be useful

Ideal rational(理性的) agent

For each percept sequence(感知序列), an ideal rational agent will act to maximise its expected performance measure, on the basis of information provided by the percept sequence plus any information built into the agent.

Rational agent doesn’t mean omniscient(全知的) or clairvoyant(先知) agent.

Percept sequence

According to the sensors, the agent can percept from the environment and get the information.

A simple agent maps each percept directly to an action. All static.

When facing a complex environment, and even dynamic, it become hard for agents to do the right thing.

Problems:

  1. what to represent
  2. How to represent
  3. interpreting the percepts

Environment

  1. Accessible&Inaccessible

An environment is accessible if the agent can obtain complete accurate and up-to-date information about all the relevant aspects.

  1. Deterministic&Non-determinist

A deterministic(确定性的) environment is one in which any action has a single guaranteed effect – there’s no uncertainty about the state that will result from performing an action.

An environment where we can quantify the non-determinism with probability is called stochastic(随机的).

  1. Episodic&non-episodic

In an episodic(情节的) environment, performance of an agent is dependent on a number of discrete episodes, with no link between the performance of an agent in different episodes.

In a non-episodic(非偶发性) or sequential environment, the current decision affects future decisions.

  1. Static & dynamic

A static environment is one that can be assumed to remain unchanged except by the performance of actions by the agent.

A dynamic environment is one that has other processes operating on it, and which hence changes in ways beyond the agent’s control.

  1. Discrete & continuous

An environment is discrete if there are a fixed, finite number of actions and percepts in it. Discrete environments could in principle be handled by a kind of “lookup table”.

Agents and Multi-Agent System

智能体与多智能体系统

Lecture 1 Introduction

Key points

  1. What’s an agent
  2. Reactive—Proactive(goal-oriented)
  3. proactive(goal-oriented)

What’s an agent

An agent is a system that:

• is situated in an environment,

• is capable of perceiving(察觉) its environment, and

• is capable of acting in its environment

with the goal of satisfying its design objectives.

Two essential features:

  1. Antonomous (decide for themselves what to do) Control over internal state and over own behaviour.
  2. Self-interested (act so as to try to achieve their goals)

Intelligent agent: can work out best way to act in complex, unpredictable environment

Want to delegate goals to the agent without having to specify exactly how those goals should be achieved. Agent works out for itself what to do to try to achieve its goals.

Definition

A computer system capable of flexible, autonomous (problem-solving) action, situated in dynamic, open, unpredictable and typically multi-agent domains.

An agent is a computer system that is capable of independent action on behalf of its user or owner (figuring out what needs to be done to satisfy design objectives, rather than constantly being told).

A multi-agent system is one that consists of a number of agents, which interact with one-another. In the most general case, agents will be acting on behalf of users with different goals and motivations.

To successfully interact, they will require the ability to cooperate, coordinate, and negotiate with each other, much as people do.

Compared with

AI:

Classical AI ignored social aspects of agency, which are important parts of intelligent activity in real-world setting.

Concurrent/Distributed Systems
  1. Agents are autonomous, capable of making independent decisions
  2. Agents are self-interested

Flexable: Reactive and Proactive

Reactivity (interact with environment changes)

A reactive system is one that maintains an ongoing interaction with its environment, and responds to changes that occur in it (in time for the response to be useful).

Proactiveness (goal-oriented)

Generating and attempting to achieve goals; not driven solely by events; taking the initiative(主动性).

Balancing reactive and goal-oriented behaviour

Reason:

Reactivity means that the system needs to be passive while proactiveness requires the system to be proactive.

Multi-agent

Agents need to be able to interact(communicates, coordinates, cooperates, competes, negotiates, delegates, persuades, incentivises …) with each other.

Social ability in agents is the ability to interact with other agents (and possibly humans) via some kind of agent-communication language, and perhaps cooperate with others.
