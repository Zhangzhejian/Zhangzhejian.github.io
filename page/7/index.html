How to do the right thing

Need to be able to

  1. identify the alternatives
  2. decide which alternative will lead to the best outcome
  3. and do so quickly enough for the decision to be useful

Ideal rational(理性的) agent

For each percept sequence(感知序列), an ideal rational agent will act to maximise its expected performance measure, on the basis of information provided by the percept sequence plus any information built into the agent.

Rational agent doesn’t mean omniscient(全知的) or clairvoyant(先知) agent.

Percept sequence

According to the sensors, the agent can percept from the environment and get the information.

A simple agent maps each percept directly to an action. All static.

When facing a complex environment, and even dynamic, it become hard for agents to do the right thing.

Problems:

  1. what to represent
  2. How to represent
  3. interpreting the percepts

Environment

  • Accessible&Inaccessible

An environment is accessible if the agent can obtain complete accurate and up-to-date information about all the relevant aspects.

  • Deterministic&Non-determinist

A deterministic(确定性的) environment is one in which any action has a single guaranteed effect – there’s no uncertainty about the state that will result from performing an action.

An environment where we can quantify the non-determinism with probability is called stochastic(随机的).

  • Episodic&non-episodic

In an episodic(情节的) environment, performance of an agent is dependent on a number of discrete episodes, with no link between the performance of an agent in different episodes.

In a non-episodic(非偶发性) or sequential environment, the current decision affects future decisions.

  • Static & dynamic

A static environment is one that can be assumed to remain unchanged except by the performance of actions by the agent.

A dynamic environment is one that has other processes operating on it, and which hence changes in ways beyond the agent’s control.

  • Discrete & continuous

An environment is discrete if there are a fixed, finite number of actions and percepts in it. Discrete environments could in principle be handled by a kind of “lookup table”.

System architecture styles

System architecture styles are formulated in terms of components, the way that components are connected to each other, the data exchanged between components and finally how these elements are jointly configured into a system.

  • Layered architectures
  • Object-based architectures
  • Data-centered architectures
  • Event-based architectures
Layered Architectures
  • Components are organised in a layered fashion where a component layer $L_i$ is allowed to call components at the underlying layer $L_{i -1}$
  • Widely adopted by communication network community
  • Control flows from layer to layer; requests go down the hierarchy whereas results flow upward
Object-based acchitectures

  • This architecture holds components loosely

  • Each component is defined as an object and components are connected through mechanisms defined in the interface (example, RPC)

Data-centered architecture
  • Web-based distributed systems are largely data-centric.
  • They rely on shared distributed file systems
  • Processes communicate through a common passive or active repository. They use shared web-based data services.
Event-based architecture
  • Processes essentially communicate through the propagation of events, which optionally also carry data. Event propagation is generally through publish/subscribe systems.(进程通过事件的传播来交流，可以携带数据。事件传播是通过发布/订阅系统来实现的)
  • Processes publish events after which middleware ensures that only processes that subscribe receive them.(只有订阅的进程才能接受到)

Client-server model

A server is a process implementing a specific service. A client is a process that requests a service from a server

Centralised architecture(中心化)
  • Clear distinction between clients and servers
  • A leader monitors the system
  • Fault tolerant and robust systems
Decentralised architecture(去中心化)
  • No monitor to observe processes
  • Useful architecture with reliable underlying(block chain) networks
  • Useful with large number of components in the system

Web proxy caching

  • To improve consistency of the system
  • Client-side caching stores frequently visited pages at browser’s cache.

  • Web proxy is another type of client-side cache that is placed between client and web server.

  • Web proxy can implement a shared cache.

  • Cooperative caching or distributed caching

Web Service

Caching

  • For the Client (browser): avoid extra network transfers
  • For the Web Server: reduce load on the server
  • For the Service Provider: reduce external traffic

Why:

  • Because of the locality of reference
  • Users tend to request the same object in succession
  • Some objects are popular: requested by many users
Local Cache(本地缓存)

Browsers include a local cache mechanism this local cache mechanism saves time and bandwidth, which increases performance.

Agents and Multi-Agent System (智能体与多智能体系统)

Lecture 1 Introduction

Key points

  1. What’s an agent
  2. Reactive—Proactive(goal-oriented)
  3. proactive(goal-oriented)

On going trends in computing:

  • Ubiquity(普遍): The continual reduction in cost of computing capability has made it possible to introduce processing power into places and devices that would have once been uneconomic.

  • Interconnection(互联): Since distributed and concurrent systems have become the norm, it is natural to think of theoretical models that portray computing as primarily a process of interaction

  • Intelligence(智能):The complexity of tasks that we are capable of automating and delegating to computers has grown steadily

  • Delegation(委托): Computers are doing more for us – without our intervention.

  • Human-orientation(面向人类): The movement away from machine-oriented views of programming toward concepts and metaphors that more closely reflect the way we ourselves understand the world.

What is an agent

An agent is a system that:

• is situated in an environment,

• is capable of perceiving(察觉) its environment, and

• is capable of acting in its environment

with the goal of satisfying its design objectives.

Two essential features/properties:
  1. Antonomous(自治的) (decide for themselves what to do) Control over internal state and over own behaviour.
  2. Self-interested(自利的) (act so as to try to achieve their goals)

Intelligent agent: can work out the best way to act in complex, unpredictable environment

Want to delegate goals to the agent without having to specify exactly how those goals should be achieved. Agent works out for itself what to do to try to achieve its goals.

Definition

A computer system capable of flexible, autonomous (problem-solving) action, situated in dynamic, open, unpredictable and typically multi-agent domains.

An agent is a computer system that is capable of independent action on behalf of its user or owner (figuring out what needs to be done to satisfy design objectives, rather than constantly being told).

A multi-agent system is one that consists of a number of agents, which interact with one-another. In the most general case, agents will be acting on behalf of users with different goals and motivations. To successfully interact, they will require the ability to cooperate, coordinate, and negotiate with each other, much as people do.

Compared with

AI:

Classical AI ignored social aspects of agency, which are important parts of intelligent activity in real-world setting.

Concurrent/Distributed Systems
  1. Agents are autonomous, capable of making independent decisions
  2. Agents are self-interested

Flexable: Reactive and Proactive

Reactivity (interact with environment changes)

A reactive system is one that maintains an ongoing interaction with its environment, and responds to changes that occur in it (in time for the response to be useful).

Proactiveness (goal-oriented)

Generating and attempting to achieve goals; not driven solely by events; taking the initiative(主动性).

Balancing reactive and goal-oriented behaviour

Reason:

Reactivity means that the system needs to be passive while proactiveness requires the system to be proactive.

  • We want our agents to be reactive, responding to changing conditions in an appropriate (timely) fashion.

  • We want our agents to systematically work towards long-term goals.

Multi-agent

Agents need to be able to interact(communicates, coordinates, cooperates, competes, negotiates, delegates, persuades, incentivises …) with each other.

Social ability in agents is the ability to interact with other agents (and possibly humans) via some kind of agent-communication language, and perhaps cooperate with others.

Summary

A computer system capable of flexible, autonomous (problem-solving) action, situated in dynamic, open, unpredictable and typically multi-agent domains.

  • Agents have control over the own state and behaviour (autonomous).

  • Agents balance reactive and proactive behaviour (flexible).

  • Agents have social abilities to interact with other agents (multiagent).

  • Agents perceive and act on their environment (situated).
