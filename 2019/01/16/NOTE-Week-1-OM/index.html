Complexity of algorithms

n denotes the size of input

  • Polynomial-time algorithm:
    • The worst-case running time is $\Theta(n^k)$
  • Exponential-time algorithm:
    • The worst-case running time is $\Theta(2^{dn})$ and $\Omega(2^{cn})$

Asymptotic(渐进的) notation $\Omega(n)$ $\Theta(n)$ and $O(n)$

  • $O(n)$: is an asymptotic upper bound
  • $Omega(n)$ is a lower bound

Mathmaticla Definition:

  • if $\lim_{n \to +\infty} \frac{f(n)}{g(n)} = 0$, then f(n) is strictly less than g(n). $f(n) = o(g(n))$
  • if $\lim_{n \to +\infty} \frac{f(n)}{g(n)} = \infty$, then f(n) is strictly larger than g(n). $f(n) = \omega(g(n))$
  • if $\lim_{n \to +\infty} \frac{f(n)}{g(n)} = c $, then f(n) is strictly equal to g(n). $f(n) =g(n)$

P, NP, NP-hard, NP-complete

  • P: problems can be solved in polynomial time
  • NP: ‘N’ denotes “non-deterministic”(非确定). Which meas these problems is non-deterministic can be solved in polynomial time currently(至少现在不确定有没有线性复杂度的算法能解决这些问题), but can be verified in polynomial time given an answer.
  • In order for further definition, we introduce a new definition here. “Reduction(归约，约化)” means a problem A can be reduced from problem B, if A can be transformed to B. Which means B is as hard as A.
  • NP-complete: NPC problem is kind of problem that:
    • is a NP problem(can be verificated polynomially)
    • all NP problem can be reduced from it
  • NP-hard: NP problems are as hard as NP problems
    • all NP problem can be reduced from it
    • but not a NP

Reference:Matrix67

Combinitional Optimisation(组合优化)

A combinatorial optimisation problem:

  • view an instance as a pair $(\mathcal(R), \mathcal(C))$
    • $\mathcal(R)$ is a finite set of configurations
    • $\mathcal(C)$ is a cost function, $\mathcal(C): \mathcal(R) \to real \; number$
  • Problem: find a confuguration with the minimum cost. That is to find a configuration config $\in \mathcal(R)$ such that the cost $\mathcal(C)(config)$ of this configuration is as small as possible.
  • Illustrations: if the size of the configuration set $\mathcal(R)$ is N, the the number of config, the subset of $\mathcal(R)$ is $2^N$. So in order to find the minum of the config, the complexity of the problem is $\Omega(2^n)$.
    • NP-hard
  • Examples: TSP, SAT
