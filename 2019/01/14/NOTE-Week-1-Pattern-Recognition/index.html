What is Pattern Recognition:

Aim is to build algorithms that can recognize useful patterns in data.

Ingredients required for pattern recognition:

  • A pattern exists

  • We can not pin it down mathematically

  • Data exists

Why is PR important

  • Patterns are everywhere
  • Helps make sense of the world
  • Data is cheep, knowledge is valuable
  • Lots of applications/opportunities

PR system

  • Sensing:
    • One or more transducers (e.g. camera, microphone, etc.) extract information from the physical world.
    • Data my also come from other sources
  • Pre-processing:
    • Data is preprocessed (e.g. segmentation, noise reduction).
    • This provides the “input data”.
  • Feature Extraction
    • Select features of the input data that:
      • preserve in-class similarity
      • discriminate between classes
  • Classification
    • Assign data to a class
  • Domain Knowledge

Input data

Data is classified as:

  • discrete (integer/symbolic valued) or continuous (real/continuous valued).
  • univariate (containing one variable) or multivariate (containing multiple variable).

Feature Extraction

Same as ML regression

Design Model

  • Data collection
    • What data to collect
    • How much
  • Feature choice
    • What features is possible to extract
    • What are useful for classification
  • Model Choice
    • what sort of classifier should be used
  • Training
    • How to use the data to define the classifier
  • Evaluation
    • Measure the error-rate or cost
    • need to avoid overfitting
