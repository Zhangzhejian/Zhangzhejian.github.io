(WI) Web Server and HTTP

Posted on 2019-01-04

HTTP

Designed for communicating enriched documents between computers

Proxy

A proxy acts on behalf of other clients and presents requests from other clients to a server. The proxy acts as a server in interacting with a client and as a client in interacting with a server.

  • client and server be separated by security intermediary (firewall).
  • Different Version of HTTP.
Gate way

A gateway is a server that appears to the client as if it were an origin server. It acts on behalf of other servers that may not be able to communicate directly with a client

  1. Client talks to gateway host as if final server
  2. Gateway passes data on to final server
  • Security intermediary:The client and server may be separated by a security intermediary such as a firewall, with the gateway on the server side of the firewall.
  • Non-HTTP server: Web browsers have built into them the capability to contact servers for protocols other than HTTP, such as FTP servers
Tunnel

Client talks to server as if tunnel was not there. Tunnel does nothing but forwards messages

Tunnels are used when there must be an intermediary system between client and server but it is not necessary for that system to understand the contents of messages. Eg. An example is a firewall in which a client or server external to a protected network can establish an authenticated connection.

HTTP Request format

  • Request Line
    • GET /URL /HTTP version
  • General Header Line
    • Connection Type : determine the connection type, persistent or not
  • Request Header Lines
    • Host: specify the web server
    • Accept: what format of file should be received
    • User-Agent: browser type

HTTP Response format

  • Status Line
    • HTTP version
    • status code
    • Description
  • General Header Lines
    • Date
    • Connection
  • Response Header line
    • Server type:
    • Accpet-ranges:
  • Entity header line
    • Content-type
    • Content-length
  • Entity Body
