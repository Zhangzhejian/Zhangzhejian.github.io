Lec 2: TCP/IP

Internet Protocol(IP) network layer

Provides a routhing function that attempts to deliver messages to the right destination

IP is an unreliable protocol: fragments might be lost, arrive out of order, or be duplicated

IP datagrame Header

Length of IP header: 20-60 bytes

Version field 4-bit(0100 or 0110) to determine the IP version(IPV4 IPV6)
Version field 4-bit(0100 or 0110) determine the IP version(IPV4 IPV6)
IHL(length) 4-bit (20-60) byte determine the length of this IP header
TOS(type of service) 8 bits(3,4,1) Specifies the quality of service requested
Total length field 16 bits (20-65535 bytes) the number of bytes of the whole ip datagram
(header plus data)
Identification Field 16 bits for reconstructing message from fragments
Flag field 3 bit “Reserved”,”don’t fragment(DF)”,”more fragments(MF)”
Fragment offset 13 bits
Time to live 8
Protocol 8 bits Transport layer protocol used(TCP for 6)
Header checksum 16 bits
Source 32 bits IP of the source
Destination 32 bits IP address of the destination
Options
Padding 0-31 make the header up to
Multiple of 32 bits
Fragment

MTU(Max transmission unit): the limit of the size of message it can carry

in order to fit the local MTU, the sender will fragment the message into small pieces

Algorithm

  1. The DF ﬂag bit is checked to see if fragmentation is allowed

    If DF is set, but the message needs to be fragmented to move onto a network with a lower MTU, an error message is sent by ICMP

  2. Based on the MTU value, the data ﬁeld is split into two or more parts

    every newly created data portion must be a multiple of 8 bytes (64 bits) except the last one

  3. Each data portion is placed in an IP datagram.

  4. Each of the fragmented datagrams is forwarded as a normal IP datagram

TCP(Transmission Control Protocol)

Key features

  • Connection oriented: implements mechanisms to setup and tear down a full duplex connection between end points
  • Reliable: implements mechanisms to guarantee error free and ordered delivery of information
  • Flow and Congestion controlled: implements mechanisms to control trafﬁc

Port and sockets

  • Port: 16-bit to identify to which application/process the message should be delivered
  • Socket: is the combination of a host’s IP address and a port number.Every communication in TCP is between two sockets, i.e. two hosts using particular ports
