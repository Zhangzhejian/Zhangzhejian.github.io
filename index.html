EJB

Enterprise Java Beans (EJBs) are a core mechanism for carrying out business logic on the server side of a J2EE-based system.

  • session beans

  • Local business interface
  • Remote home interface
    • Create()
    • Remove()
    • Find()
  • Local home interface

EJB life cycle

CMP(container-managed persistence) is an automated synchronisation of the entity bean data and the actual stored data in the resource tier provided by J2EE.

Web services

    • Intercepting ﬁlter

      • deﬁnes a structure of pluggable ﬁlters to add pre and post-processing of web requests/responses, e.g.: for security checking.

      • deﬁnes a single point of access for web system services, through which all requests pass. Enables centralised handling of authentication, etc.

    • Composite view

      • uses objects to compose a view out of parts (subviews)

    • Value object

      • Purpose: improve the efficiency of accessing persistent data(entity beans) by grouping data and transferring data as a group of attribute values of each object
      • 去除了JAVA bean 中的get和set方法，加快读取

    • Session facade

      • uses a session bean as facade to hide complex interactions between business objects in one workﬂow/use case.

    • Composite entity

      • Uses entity beans to manage a set of interrelated persistent objects to improve efficiency.

    • Data access object(DAO)

      • provides an abstraction of persistent data source ac-

        cess

    • Value list handler

      • This pattern provides a means to access result lists element by element
      • It provides an iterator-style interface allowing navigation of such lists

Design patterns

EIS(Enterprise information system)

  • An enterprise information system (EIS) is a software system which holds core business data for a company or organisation and which performs operations involving this data.
  • usually large and complex systems(Distributed)
  • be used by different applications as a resource

EIS architecture and components

  1. Client tier
    • responsible to display information and receive information
    • HTML
  2. Presentation tier
    • Managing presentation of information
    • controlling interaction sequence
    • Relaying user requests
    • JSP
  3. Business tier
    • Components implementing the business rules and data
    • Beans
  4. Integration tier
    • Mediates between the business and resource tiers.
    • Manages data retrieval and data conversion.
    • Insulates higher tiers from direct knowledge of low-level data concepts.
    • DB Interfaces
  5. Resource tier
    • Contains persistent data storage and external services.
    • DB

EIS Business tier design

Entity beans vs session beans

  • Session bean
    • Dedicated to a single client
    • Live only for the duration of the client’s session
    • Not persistent
    • Used to model stateful or stateless interactions between the client and business tier components.
    • A session bean encapsulates a group of operations, usually corresponding to the use cases of the system, which operate on one or more of the entities of the system.
    • Each use case of the system would normally be implemented by a session bean
    • 一个对一个client，生存周期为会话的周期，非持久，用来描述client和应用层组件的交互，每个用例通常都可以用一个session bean表示。
  • Entity bean
    • Provides an object view of persistent data
    • Coarse-grained
    • Multi-user and Long-lived
    • Entity beans act as an object-oriented facade for data of system, which may actually be stored as relational tables, or as XML datasets
    • Normally each entity from the PIM class diagram of the system will be managed by some entity bean.
    • 持久的数据，粗粒度，多用户长生存周期。entity bean是系统数据的对象化表示，PIM类图中的每个实体都可用其表示

Stateless session bean vs Stateful session bean

  • Stateless
    • Provide a service by a single method call
    • Instantaneous
    • can be shared and reused
  • Stateful
    • Provide a service specific to a single client
    • cannot be shared
    • Involve several related operations

EIS design issues

  • Data security
    • Password encryption
    • HTTPS
  • Separating presentation, business logic and data processing code
    • EIS components: session beans for business processing, entity beans for complex bbusiness data
    • Separate tiers
  • Pooling database connections
    • Database connection pool

Introduction

  • Generate a collection of “weak” classiﬁers from a given dataset.
  • Each week classifier is a simple classifier, e.g., a half-space classifier
  • Combine classifiers to be a final one (averaging or voting)

Bagging(bootstrap aggregating)

  • Uses multiple versions of a training set.
  • Each created by drawing $n’ < n$ samples from $\scr D$ with replacement.(有放回的)
  • Each bootstrap data set is used to train a different weak classiﬁer.
  • Final classiﬁcation decision is based on the vote of each weak classiﬁer.

Procedure

  1. Start with dataset$\scr D$

  2. Generate M dataset $ \scr D_1, \scr D_2, \dots ,\scr D_M$

  3. Learn weak classifier for each dataset

    • $f_i(x)$ for dataset $\scr D_i$

  4. Combine all weak classifiers using a majority voting scheme

Boosting

Goal: to improve the accuracy of any given learning algorithm

Adaboost(Adaptive Boost)

  • Advantages
    • Simple for implementation
    • Good classification accuracy and generalisation
    • Can be applied to different classifiers
  • Disadvantage
    • Solution is suboptimal(次优)
    • sensitive to noisy data and outliers(异常点)

Algorithm:

Ref:

  1. http://www.52caml.com/head_first_ml/ml-chapter6-boosting-family/
  • First create a classiﬁer with accuracy on the training set greater than average.
  • Then add new component classiﬁers to form an ensemble whose joint decision rule has arbitrarily high accuracy on the training set.

Stacked generalization

  • Two levels(Input and output)
  • The input will be processed by a group of classiﬁers in the ﬁrst level
  • The output from the ﬁrst layer will go though the classiﬁer in the second level to make the ﬁnal decision
  1. Prepare K sub-training dataset: $\scr D_1 , \scr D_2 , ··· , \scr D_K$ of different size and samples
  2. Train the k th classiﬁer using training dataset $\scr D_k $
  3. Create a new training dataset for the second-level classiﬁer. The outputs of the ﬁrst-level classiﬁers ($z_1 , z_2 , ··· , z_K$ ) and the target output (y) of the classiﬁcation problem (supervised) are taken to form the new dataset $D_{new}$
  4. Train the second-level classiﬁer using the dataset $D_{new}$
