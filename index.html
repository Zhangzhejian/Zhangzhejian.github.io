Argument game approach(ground extension)

Difference between game tree and strategy

A game tree represents all the possible games that can be played according to the rules of the game:

• Every branch is a dispute (sequence of allowable moves).

• The game tree contains all possible disputes.

A strategy for player p is a partial game tree:

• Every branch is a dispute (sequence of allowable moves).

• The tree only branches after moves by p.

• The children of p’s moves are all the legal moves by the other player.

A strategy S for player p is winning if and only if p wins all disputes in S.

Decomposition approach(prefered extension)

Idea is to decompose an argumentation framework into a set of sub-frameworks and compute the semantics of each subframework locally.

We need to perform the decomposition so that the resulting subframeworks respect the dependency of the attack relation and form an acyclic graph(非循环图).

Strongly Connected components(SCC):

Two arguments A and B are in the same strongly connected component if and only if there is a path from A to B and a path from B to A (using the attack relation).

The grounded extension is the minimal complete extension with respect to set inclusion (ie, the minimal subset of the set of complete extensions).

A preferred extension is a complete extension that is maximal with respect to set inclusion (ie, a maximal subset of the set of all complete extensions).

Lecture 5: Communication, coordination and working together

Agent communication

agents are autonomous and self-interested. One agent can’t force another to perform some action

Instead they can perform communicative actions to try to influence other agents.

Speech act theory

Speech act theory treats communication as action. It is a theory of how utterances are used to achieve intentions.

Three aspects of speech acts

  • The locutionary act(语言行为): the act of making the utterance

    • Saying “Please make me some tea”

  • The illocutionary act(言外行为): the conveying of the speaker’s intentions to the hearer

    • The speaker has requested the hearer to make some tea

  • The perlocution(表达效果): the effect of the act

    • The hearer (perhaps) makes some tea.

Speaker makes a locutionary act(语言行为) with hope(言外行为) that the hearer correctly recognises the speaker’s intentions and acts(表达效果) accordingly.

we can in general think of a speech act as having two components:

  • a performative verb (e.g request, inform, promise)
  • Propositional content命题内容(e.g ‘the door is closed’)

Example：

performative = request content = ‘the door is closed’ speech act = ‘please close the door’

performative = inform content = ‘the door is closed’ speech act = ‘the door is closed!’

performative = inquire content = ‘the door is closed’ speech act = ‘is the door closed?’

Plan-based theory of speech acts

request(s, h, φ)

pre:

s believes (h can do φ)

s believes (h believes (h can do φ))

s believes (s want φ)

post:

h believe (s believe (s want φ))

inform(s, h, φ)

pre:

s believes φ

s believes (s wants (h to believe φ))

post:

h believes (s believes φ)

KQML(Knowledge Query Manipulation Language) Agent-based

comprised of two parts:

  1. the KQML. ‘Outer’ language that defines ‘communicative verbs’ or performatives, examples:
  • ask-if (‘is it true that. . . ’)
  • perform (‘please perform the following action. . . ’)
  • tell (‘it is true that. . . ’)
  • reply (‘the answer is . . . ’)
  1. The knowledge interchange format (KIF).
  • Language for expressing message content. Used to state:
    • Properties of things in a domain
      • E.g. ‘The temperature of m1 is 83 Celsius’ (= (temperature m1) (scalar 83 Celsius))
    • Relationships between things in a domain
      • E.g. ‘Any individual with the property of being a person also has the property of being a mammal’ (defrelation person (?x) :=> (mammal ?x))
    • General properties of things in a domain
      • E.g. ‘An object is a bachelor if the object is a man and is not married’ (defrelation bachelor (?x) := (and (man ?x) (not (married ?x))))
Criticisms:

  • Semantics of performatives only informally defined, so open to different interpretations.(执行动词的定义是非正式的，所以可以会导致不同的解释)

  • No ‘commissives’, could argue that performative set rather ad hoc.

working together

It is often necessary or preferable for agents to work together in order to achieve their goals.

Since agents are autonomous they make decisions about what to do at run-time, they must be capable of dynamically coordinating their activities and cooperating with others.

Compare this with traditional distributed and concurrent systems, where coordination and cooperation is typically hardwired at design time.

two criteria for assessing how well agents work together.

Coherence(连贯性)

is about how “well the [multiagent] system behaves as a unit along some dimension of evaluation”. 作为一个整体的表现

We can measure it in terms of solution quality, e.g. how efficiently resources are used, conceptual clarity.

Coordination(协调性)

is about “the degree. . . to which [the agents]. . . can avoid “extraneous” activity [such as] . . . synchronizing and aligning their activities”. 对于各自活动的同步和调整

If the system is perfectly coordinated, agents will not get in each others’ way, in a physical or a metaphorical sense.

Solutions to individual subproblems are integrated into an overall solution

Lecture 5: Strategic Reasoning(Game Theory)

Game Theory

Game theory is a framework for analysing interactions between a set of agents.

Key points in game theory:
  • Dominant strategy
  • Nash equilibrium strategy
  • Pareto optimal strategies
  • Strategies that maximise social welfare
Dominant strategies(占优策略)
Nash Equilibrium(纳什均衡)

In general, we will say that two strategies s 1 and s 2 are in Nash equilibrium (NE) if:

  1. under the assumption that agent i plays s_1 , agent j can do no better than play s_2 ; and

  2. under the assumption that agent j plays s_2 , agent i can do no better than play s_1 .

  • Not every interaction scenario has a pure strategy NE.
  • Some interaction scenarios have more than one NE.
Pareto optimal Strategy

An outcome is said to be Pareto optimal (or Pareto efﬁcient) if there is no other outcome that makes one agent better off without making another agent worse

Social welfare

The social welfare of an outcome ω is the sum of the utilities that each agent gets from ω

Lecture 4 More on agent architectures

key points:

  1. deductive reasoning agents

Deductive reasoning(演绎推理) agents

Deductive reasoning = particular symbolic approach where representations are logical formulas and reasoning used is logical deduction(逻辑演绎) (theorem proving).

  • Symbolic AI paradigm

  • Contains an explicitly represented, symbolic model of the world.

  • Makes decisions (for example about what actions to perform) via symbolic reasoning.

Use logic to encode a theory stating the best action to perform in any given situation.

Let:

• ρ be this theory (typically a set of rules)

• Δ be a logical database that describes the current state of the world

• Ac be the set of actions the agent can perform

• Δ |- ρ ϕ means that ϕ can be deduced from Δ using ρ

according to the input of environment and the logical database get the current state. then based on the theory and the current state, the agent get the right decision.

planning agents

Planning agents find a sequence of actions that transforms an initial state into a goal state.

Planning involves issues of both Search and Knowledge Representation.

Reactive agent architectures

Traditional AI View

• Symbolic (internal) representation.

• Manipulate this representation to work out what to do next.

Alternative View

• Symbolic reasoning is very resource demanding.

• Effective behaviour does not need representation and manipulation.

• Intelligent behaviour is a product of the agent’s interaction with its environment.

• Intelligent behaviour emerges from the interaction of various simpler behaviours.
