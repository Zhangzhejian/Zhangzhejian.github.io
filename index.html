(OM)Network Flow Problems

Posted on 2019-02-12

Flow network

  • $V$ – set of n nodes,$ E $– set of m directed edges (links);
  • for each $(u, v) \in E, c(u, v) \geq 0$ is the capacity of edge $(u, v)$;

  • two distinguished nodes: source $s$ and sink $t$. $(s \ne t)$

  • Aim:

    • find a maximum flow from the source to the sink:
    • a ﬂow of the maximum total amount, while the ﬂow on each edge is not greater than the capacity of this edge;
    • a continuous ﬂow of the maximum total rate, while the rate of ﬂow on each edge is not greater than the capacity of this edge.

Flows-Feasibility Problem(Transshipment problem)

Definition of the value of flow

Assume:

  • If $(u,v) \in E$ ,then $(v,u) \in E$

  • A flow is a function $f: E \to R$, where $f(u,v) \geq 0$ is the flow on edge that has the following properties:

    • Capacity constrants:

      • $0 \leq f(u,v) \leg c(u,v)$

    • Flow conservation:(对于除了原点和终点之外的点)

      • For each node $v \in V-{s,t}$, the total flow into $v$ is equal to the total flow out of $v$

    • For each edge $(u,v) \in E$, if $f(u,v) >0$, then $ f(v,u) =0$

The value of a ﬂow $f$ is the net ﬂow into the sink (which is equal to the net ﬂow from the source, due to the ﬂow conservation condition):

Failover

either all failures must be failstop, or the system must be synchronous

Properties of Primary Backup

Primary Backup suffers a single point of failure in the primary manager.

  • At any one time there must be at most one primary replication manager
  • Each front end maintains a reference to the manager it currently believes to be primary
  • A backup server ignores any client requests

Failover time is the period for which there is no primary replciation manager

The primary manager receiving request RAi from a client:

  1. Update its state
  2. Forwards the request to backup
  3. replies to client that sent RAi

this period is called blocking time

Consensus(达成一致)

Acheiving consensus for Choosing a new Primary Manager

Consensus Protocol Properties

  • Termination: every correct process will decide on a value
  • Validity: if all proposals are for v then all correct processes eventually decide v
  • Agreement: if one correct process decides v then all correct processes eventually decide v
  • Integrity: every correct process decides at most one v, and either v = FAIL or some process must have proposed
